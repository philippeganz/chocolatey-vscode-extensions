## Overview

Visual Studio Code extensions are highly sought after by developers in enterprise environments. However, public Chocolatey packages for VS Code extensions often rely on the `code --install-extension` CLI command. This command forces the local machine to reach out to the internet to download the `.vsix` payload, which completely fails in offline, proxy-restricted, or air-gapped environments.

This repository solves that. We maintain **"Pure Packages"** where the `.vsix` payload is automatically downloaded by the AU script and fully embedded *inside* the Chocolatey `.nupkg`.

> [!NOTE]
> **What is AU?** The Chocolatey Automatic Updater (AU) is a community framework that runs on a daily schedule to detect upstream software updates, rebuild the packages, and automatically publish them to the public gallery without human intervention. This repository serves as the backend factory machinery powering those updates.

## Setup

### What these packages affect

When you install an extension from this repository, it affects your local VS Code environment:

- The raw `.vsix` extension payload is natively unpacked into your user profile (`%USERPROFILE%\.vscode\extensions`).
- Chocolatey registers the installation so it can be cleanly uninstalled or upgraded via CLI.
- No OS-level services or firewall rules are modified by default.

### Setup requirements

You must have [Chocolatey](https://chocolatey.org/) installed on your machine.
If the extension requires a native OS toolchain (e.g., Python, Git), Chocolatey will attempt to install it as a dependency.

## Usage

If you are a standard user looking to install these extensions on your machine, **you do not need to clone this repository or build any scripts**.

Simply install the packages directly from the Chocolatey Community Repository using the standard CLI:

```powershell
choco install vscode-<extension-name>
```

> [!TIP]
> **Naming Convention**: All packages drop the publisher namespace and are universally prefixed with `vscode-`. For example, the extension `ms-vscode.remote-server` is packaged as `vscode-remote-server`.

All generated packages are completely self-contained, offline-first, and natively air-gap ready.

## The Air-Gap Problem

When you install an extension from this repository:

1. No internet connection is required at install time.
2. Extensions are mapped 1:1 to their Chocolatey dependencies.
3. Metadata, `README`, and `LICENSE` files are extracted directly from the `.vsix` archive ensuring perfect upstream compliance.
4. Deep scanning is employed to ensure packages do not invoke hidden runtime network triggers (e.g., `wget`, `npm install`).
5. Native AU checksum generation is explicitly bypassed (`-ChecksumFor none`) because the `.vsix` is bundled inside the `.nupkg`. The payload is mathematically protected by the `.nupkg`'s native SHA512 hash generated by the Community Repository.

### How to Add a New Extension

We welcome community contributions! Because we use an automated factory, adding a new extension requires zero manual templating. You have two options:

**Option A: The Fast Route (Open an Issue)**
If you don't want to mess with code, simply open a **New Package Request** issue.

1. Go to the VS Code Marketplace and find your extension.
2. Look for the **Unique Identifier** (e.g., `ms-python.python`) on the right sidebar.
3. Open an issue with that identifier. Our maintainers will trigger the Factory and roll out the package for you!

**Option B: The Contributor Route (Pull Request)**
If you want to scaffold the package yourself:

1. Create a new branch: `feature/add-my-extension`.
2. Find the **Unique Identifier** (e.g., `Publisher.ExtensionName`) on the VS Code Marketplace.
3. Use the Pool Manager to automatically scaffold the package:

   ```powershell
   .\bin\Manage-ExtensionPool.ps1 -Add "Publisher.ExtensionName"
   ```

4. The script will automatically pull the binaries, update the configuration, and drop the fully generated package into the `automatic/` directory.
5. Commit the resulting changes and open a Pull Request!

Once an extension is merged into the `main` branch, the `au-updater.yml` GitHub Action takes over. Every 6 hours, it crawls the repository and automatically builds and pushes updates to the Chocolatey Gallery.

## Limitations

- Extensions that require OS-level toolchains (e.g., Python, Git) will list them as separate Chocolatey dependencies (Meta-Packages).
- This repository strictly targets the `win32-x64` architecture (and universal extensions).
